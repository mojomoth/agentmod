কোডিং এজেন্টদের জন্য প্রজেক্ট-ভিত্তিক বিচ্ছিন্নতা এবং হস্তান্তর।

`agentmod` **Claude Code**, **Codex CLI**, এবং **OpenCode** এর কনফিগারেশন, স্কিল, প্লাগইন, সেশন, ক্যাশ এবং কর্মিং কন্টেক্স্ট আপনি যেটি প্রজেক্টে কাজ করছেন সেটির ভিতরে রাখে — এবং সেই পরিবেশটি একটি স্ন্যাপশটে প্যাক করে যা আপনি অন্য মেশিনে হস্তান্তর করতে পারেন।

এটি দুটি ভূমিকা পালন করে:

1. **এজেন্ট হোম রাউটার।** `.agentmod/agentmod.toml` সম্বলিত একটি ডিরেক্টরি ট্রি এর ভিতরে, একটি শেল হুক প্রতিটি এজেন্টের হোমকে `.agentmod/` এ রুট করে। বাইরে, প্রতিটি ভেরিয়েবল ঠিক যেমন ছিল তেমন পুনরুদ্ধার করা হয় এবং আপনার গ্লোবাল সেটআপ অস্পর্শিত থাকে।
2. **হস্তান্তর সরঞ্জাম।** `agentmod handoff create` `.agentmod/` কে একটি যাচাইযোগ্য `.amod` স্ন্যাপশটে প্যাক করে (অথবা, `--for-git` সহ, `.agentmod-handoff/` এর অধীনে একটি প্রতিশ্রুতিযোগ্য ফাইল ট্রি)। **Git আপনার সোর্স সরায়; agentmod এজেন্ট পরিবেশ সরায়।**

## agentmod কী নয়

- **Docker স্যান্ডবক্স নয়।** এটি আপনার নিজের শেলে পরিবেশ ভেরিয়েবল রুট করে। কোন কন্টেইনার, কোন ভিএম, কোন সিসকল ফিল্টারিং নেই।
- **সম্পূর্ণ নিরাপত্তা বিচ্ছিন্নতা নয়।** একটি সরঞ্জাম যা রুট করা ভেরিয়েবলগুলিকে উপেক্ষা করে এখনও আপনার গ্লোবাল হোমে পৌঁছাতে পারে। Claude Bash গার্ড (নিচে) গভীরতায় প্রতিরক্ষা, নিরাপত্তা সীমানা নয়।
- **শিম নয়।** এটি কখনও `claude`, `codex`, বা `opencode` কমান্ডগুলিকে বাধা দেয় বা মোড়ায় না। আপনি সেগুলি সরাসরি, অপরিবর্তিতভাবে চালাতে থাকেন।
- **HOME পরিবর্তনকারী সরঞ্জাম নয়।** `HOME` কখনও পুনর্নির্ধারিত হয় না।
- **সোর্স-কোড ব্যাকআপ সরঞ্জাম নয়।** স্ন্যাপশটগুলি ডিফল্টরূপে আপনার সোর্স কোড কখনও অন্তর্ভুক্ত করে না। সোর্সের জন্য git ব্যবহার করুন।

## এটি কীভাবে কাজ করে

`agentmod hook zsh` / `agentmod hook bash` একটি ছোট স্ব-নিহিত শেল ফাংশন প্রিন্ট করে (`agentmod init` দ্বারা আপনার rc ফাইলে ইনস্টল করা)। প্রতিটি প্রম্পট এবং ডিরেক্টরি পরিবর্তনে এটি `.agentmod/agentmod.toml` খুঁজতে উপরে হাঁটে:

- **একটি প্রজেক্টে প্রবেশ করা** বর্তমান মান সংরক্ষণ করে এবং নির্ধারণ করে:

  | ভেরিয়েবল | রুট করা হয়েছে |
  |---|---|
  | `CLAUDE_CONFIG_DIR` | `.agentmod/claude` |
  | `CODEX_HOME` | `.agentmod/codex` |
  | `OPENCODE_CONFIG` | `.agentmod/opencode/opencode.json` |
  | `NPM_CONFIG_PREFIX` | `.agentmod/node` |
  | `NPM_CONFIG_CACHE` | `.agentmod/node/npm-cache` |
  | `PNPM_HOME` | `.agentmod/node/pnpm` |
  | `BUN_INSTALL` | `.agentmod/node/bun` |
  | `XDG_CONFIG_HOME` / `XDG_DATA_HOME` / `XDG_CACHE_HOME` / `XDG_STATE_HOME` | `.agentmod/opencode/xdg/…` — **শুধুমাত্র** `opencode.xdg_full_isolation = true` সহ |

  `PATH` ঠিক একটি এন্ট্রি অর্জন করে, `.agentmod/node/bin` (রুট করা প্রিফিক্সের অধীনে npm এর গ্লোবাল বিন)। বুককিপিং ভেরিয়েবলগুলি (`AGENTMOD_ACTIVE`, `AGENTMOD_PROJECT_ROOT`, `AGENTMOD_ROOT`, `AGENTMOD_VARS`, `AGENTMOD_SAVED_*`) কী পূর্বাবস্থায় ফিরিয়ে আনতে হবে তা রেকর্ড করে।

- **প্রজেক্ট ছেড়ে যাওয়া** প্রতিটি সংরক্ষিত মান পুনরুদ্ধার করে এবং `PATH` এন্ট্রি স্ট্রিপ করে — একটি নিখুঁত বিপরীত। দুটি agentmod প্রজেক্টের মধ্যে সরাসরি সুইচ করা একটি ধাপে পুনরুত্থার করে কোন প্রজেক্টের পাথ ফাঁস না করে।

`agentmod.toml` এ প্রতিটি এজেন্টের জন্য রুটিং বন্ধ করা যেতে পারে (`claude.enabled`, `codex.enabled`, `opencode.enabled`, `node.enabled`)।

## ইনস্টল

যেটি আপনার সেটআপের সাথে খাপ খায় তা নির্বাচন করুন — প্রতিটি একই একক বাইনারি ইনস্টল করে:

```sh
# npm (আপনার প্ল্যাটফর্মের জন্য প্রাক-নির্মিত বাইনারি ইনস্টল করে)
npm install -g agentmod

# ইনস্টল স্ক্রিপ্ট (ম্যাচিং রিলিজ ডাউনলোড করে, sha256 যাচাই করে)
curl -fsSL https://raw.githubusercontent.com/mojomoth/agentmod/main/install.sh | sh

# go install (Go টুলচেইন প্রয়োজন)
go install github.com/mojomoth/agentmod@latest
```

অথবা সোর্স থেকে বিল্ড করুন (Go 1.26+, একমাত্র মডিউল নির্ভরতা `BurntSushi/toml`):

```sh
git clone https://github.com/mojomoth/agentmod && cd agentmod
go build -o agentmod .
# বাইনারিটি আপনার PATH এর কোথাও রাখুন
```

## দ্রুত শুরু

```sh
cd ~/work/myproject
agentmod init          # .agentmod/ তৈরি করে, .gitignore সম্পাদন করে, 
                       # আপনার rc ফাইলে শেল হুক ইনস্টল করে, 
                       # প্রমাণীকরণ অনুলিপি করার অফার দেয়
# প্রথমবার শুধুমাত্র: এই শেলে হুক এখনও সক্রিয় নয় —
# একটি নতুন টার্মিনাল খুলুন, বা: exec $SHELL

cd ~/work/myproject    # হুক সক্রিয় হয়; এটি পরীক্ষা করুন:
agentmod status        # "AgentMod: active", রুট করা পাথ তালিকাভুক্ত
claude                 # সাধারণ কমান্ড — এখন প্রজেক্ট-স্থানীয় হোম ব্যবহার করছে
agentmod install gstack   # প্রজেক্ট-স্থানীয় দক্ষতা, গ্লোবাল হোম অস্পর্শিত

agentmod pack          # স্ন্যাপশট .agentmod/snapshots/<name>-<stamp>.amod এ
agentmod doctor        # যেকোনো সময় শুধুমাত্র পড়া নির্ণয়
```

প্রাপ্তি মেশিনে:

```sh
cd ~/work/myproject    # উৎস git এর মাধ্যমে এসেছে
agentmod init
agentmod unpack myproject-20260611-123045.amod
# মুদ্রিত পুনরায় লগইন নোট অনুসরণ করুন; ডাক্তার স্বয়ংক্রিয়ভাবে চলে
```

## `agentmod init`

বিপরীত — পুনরায় চালানো পূরণ করে যা কোনটি অনুপস্থিত এবং কখনও বিদ্যমান `agentmod.toml` বা কোন ব্যবহারকারী ফাইল অপ্রতিদ্বন্দ্বী করে না। এটা:

- `.agentmod/{claude,codex,opencode,node,snapshots,logs}` এবং একটি ডিফল্ট `agentmod.toml` তৈরি করে;
- Claude Bash গার্ডকে `.agentmod/claude/settings.json` এ যুক্ত করে;
- `.agentmod/` এ `.gitignore` যোগ করে (শুধুমাত্র একটি গিট রেপোজিটরির ভিতরে তৈরি);
- শেল হুক `~/.zshrc` বা `~/.bashrc` এ একটি বেষ্টিত ব্লক হিসাবে ইনস্টল করে ($SHELL থেকে আপনার শেল; ব্লক জায়গায় আপডেট করা হয়, কখনও ডুপ্লিকেট নয়, এবং আপনার নিজের rc সামগ্রী কখনও স্পর্শ করা হয় না);
- বিদ্যমান Claude/Codex প্রমাণীকরণ ফাইলগুলি প্রজেক্ট-স্থানীয় হোমে **অনুলিপি** করার অফার করে (নিচে "প্রমাণীকরণ" দেখুন) — অনুলিপি শুধুমাত্র একটি স্পষ্ট `y` এ ঘটে।

পতাকা: `--no-shell-hook` সমস্ত rc-ফাইল সম্পাদন এড়িয়ে যায়; `--yes` / `--non-interactive` কখনও প্রম্পট করে না এবং তাই কখনও প্রমাণীকরণ অনুলিপি করে না (CI এর জন্য)।

## সাধারণ `claude`, `codex`, `opencode` ব্যবহার করা

কোন মোড়ক কমান্ড নেই। একটি সক্রিয় প্রজেক্টের ভিতরে সাধারণ কমান্ডগুলি সহজে রুট করা হোমগুলি দেখে:

- **Claude Code** `CLAUDE_CONFIG_DIR` পড়ে → প্রজেক্ট-স্থানীয় সেটিংস, ব্যবহারকারী-স্তরের দক্ষতা/প্লাগইন, সেশন, ইতিহাস। (প্রজেক্ট-স্তরের `.claude/` *সর্বদা* নেটিভভাবে পড়া হয় — সীমাবদ্ধতা দেখুন।)
- **Codex CLI** `CODEX_HOME` পড়ে → প্রজেক্ট-স্থানীয় `config.toml`, `auth.json`, সেশন, ইতিহাস, লগ।
- **OpenCode** `OPENCODE_CONFIG` পড়ে → প্রজেক্ট-স্থানীয় কনফিগ ফাইল। এটি *আংশিক* বিচ্ছিন্নতা ডিফল্টরূপে — সীমাবদ্ধতা দেখুন।

### প্রমাণীকরণ

তাজা প্রজেক্ট-স্থানীয় হোম প্রমাণপত্র ছাড়াই শুরু হয়:

- **macOS এ Claude**: কিছু করার প্রয়োজন নেই — প্রমাণপত্র Keychain এ বাস করে এবং প্রতিটি কনফিগ ডিরের সাথে ভাগ করা হয় (যার অর্থ এটি প্রজেক্ট প্রতি বিচ্ছিন্ন *নয়*)।
- **Linux/Windows এ Claude**: প্রজেক্টের ভিতরে `claude login` চালান, অথবা init এর অফার স্বীকার করুন `~/.claude/.credentials.json` অনুলিপি করার।
- **Codex**: প্রজেক্টের ভিতরে `codex login` চালান, অথবা init এর অফার স্বীকার করুন `~/.codex/auth.json` অনুলিপি করার।

প্রমাণপত্র ফাইলগুলি **কখনও স্ন্যাপশটে ভ্রমণ করে না** (নামে বাদ দেওয়া, নির্বিশেষে তারা কীভাবে সেখানে এসেছে)।

## gstack ইনস্টলেশন

[gstack](https://github.com/garrytan/gstack) এর ইনস্টলার `~/.claude/skills/gstack` এ হার্ডকোড করে — ঠিক বৈশ্বিক দূষণ যা agentmod বিদ্যমান রাখতে চায় প্রতিরোধ করতে। সুতরাং:

```sh
agentmod install gstack            # .agentmod/claude/skills/gstack এ ক্লোন করুন
agentmod install gstack --force    # বিদ্যমান প্রজেক্ট-স্থানীয় ইনস্টল প্রতিস্থাপন করুন
```

ইনস্টলার git এর সাথে ক্লোন করে, কখনও gstack এর নিজস্ব সেটআপ স্ক্রিপ্ট চালায় না, এবং `~/.claude/skills` এর তালিকা — এবং পরবর্তী ক্রম চেক করে। যেকোনো পরিবর্তন বৈশ্বিক ডিরেক্টরিতে একটি লঙ্ঘন হিসাবে রিপোর্ট করা হয় এবং কমান্ড ব্যর্থ করে। `agentmod doctor` আলাদাভাবে সতর্ক করে যখনই একটি *বৈশ্বিক* gstack ইনস্টল বিদ্যমান থাকে (এমনকি একটি যা আপনি agentmod গ্রহণের আগে নিজে ইনস্টল করেছিলেন), কারণ বৈশ্বিকভাবে ইনস্টল করা দক্ষতাগুলি প্রতিটি প্রজেক্টে ফাঁস করে।

## হস্তান্তর (`.amod` স্ন্যাপশট)

```sh
agentmod handoff create [--output PATH] [--allow-findings] [--allow-dirty]
agentmod handoff list
agentmod handoff inspect FILE      # ম্যানিফেস্ট + সম্পাদন রিপোর্ট, কোন আহরণ নয়
agentmod handoff verify  FILE      # প্রতিটি সদস্য পুনরায় হ্যাশ করুন; মিসম্যাচে 3 এ প্রস্থান করুন
agentmod handoff restore FILE      # .agentmod/ প্রতিস্থাপন করুন (ব্যাকআপ প্রথমে নেওয়া হয়)
agentmod pack / agentmod unpack    # তৈরি / পুনরুদ্ধার করার উপনাম
```

একটি স্ন্যাপশট একটি জিপ যার ছয়টি রুট সদস্য রয়েছে — `manifest.json`, `inventory.json` (প্রতিটি ফাইল আকার/sha256/মোড), `REDACTION.md` (কী বাদ দেওয়া হয়েছে এবং কেন, প্লাস গোপন-স্ক্যান ফলাফল), `HANDOFF.md` এবং `RESTORE.md` (প্রাপকের জন্য মানব নির্দেশনা), `checksums.txt` (`shasum -a 256 -c`-সামঞ্জস্যপূর্ণ) — এবং `payload/.agentmod/…` এর অধীনে পেলোড। সৃষ্টি পরমাণুবাদী এবং নির্ণায়ক; ম্যানিফেস্ট গিট শাখা/প্রতিশ্রুতি/নোংরা স্থিতি রেকর্ড করে কোনো প্রমাণপত্র রিমোট ইউআরএল থেকে স্ট্রিপ করে। একটি নোংরা কর্মট্রি `--allow-dirty` না হলে প্যাক করতে অস্বীকার করে।

`inspect` এবং `verify` যেকোনো জায়গায় কাজ করে — প্রাপক কোনো প্রজেক্ট সেটআপ করার আগে একটি স্ন্যাপশট অডিট করতে পারে।

### গোপনীয়তা বর্জন নীতি

দুটি স্তর, উভয়ই ডিফল্টরূপে:

1. **বর্জন নিয়ম** পেলোড থেকে পরিচিত-সংবেদনশীল ফাইলগুলি ড্রপ করে এবং প্রতিটি `REDACTION.md` তে তালিকাভুক্ত করে: প্রমাণপত্র ফাইলগুলি নামের দ্বারা (`.credentials.json`, `auth.json`, `credentials*`), `*.env` / `.env.*`, SSH কীগুলি (`id_*`, `*.pem`, `*.pub`), শংসাপত্র ডিরেক্টরিগুলি (`.ssh`, `.aws`, `.azure`, `.gcloud`, `.kube`, `.gnupg`, `.docker`), কীচেইন ফাইলগুলি, `.git`, `node_modules`, ক্যাশ এবং অস্থায়ী ডিরেক্টরিগুলি।
2. **প্রতিটি *রাখা* ফাইলের উপর সামগ্রী স্ক্যান।** ব্যক্তিগত-কী উপাদান সৃষ্টি সম্পূর্ণভাবে অস্বীকার করে যদি আপনি `--allow-findings` পাস না করেন (এবং তারপর `REDACTION.md` এ HARD হিসাবে চিহ্নিত করা হয়)। সম্ভাব্য টোকেনগুলি (AWS অ্যাক্সেস কী আইডি, GitHub টোকেন, `sk-…` কী, `api_key=`-শৈলী বরাদ্দ) সতর্ক করা হয় কিন্তু ব্লক করে না।

স্ক্যান হিউরিস্টিক। **একটি স্ন্যাপশট ভাগ করার আগে `REDACTION.md` (অথবা `handoff inspect`) পর্যালোচনা করুন** — সেশন এবং কর্মিং প্রসঙ্গ ডিজাইন দ্বারা ভ্রমণ করে এবং আপনি একটি এজেন্ট কথোপকথনে পেস্ট করা কিছু উদ্ধৃত করতে পারেন। স্ন্যাপশটগুলি এই কারণে 0600 মোডে লেখা হয়; তাদের ব্যক্তিগত ফাইলের মতো আচরণ করুন।

## Git হস্তান্তর

```sh
agentmod pack --for-git    # .agentmod-handoff/ প্রজেক্ট রুট এ লেখে
git add .agentmod-handoff && git commit
```

`.amod` এর মতো একই ছয়টি সদস্য এবং পেলোড, কিন্তু সাধারণ ফাইলের একটি প্রতিশ্রুতিযোগ্য ট্রি হিসাবে (`shasum -a 256 -c checksums.txt` ডিরেক্টরিতে কাজ করে)। ডিফল্ট বর্জনগুলির উপরে এটি সমস্ত তিনটি এজেন্টের **সেশন, প্রতিলিপি, ইতিহাস এবং লগ** স্ট্রিপ করে — তারা নিয়মিত পেস্ট করা গোপনীয়তা ধারণ করে এবং একটি সংগ্রহস্থলে আসে না। ভাগ করার জন্য নিরাপদ কর্মিং প্রসঙ্গ (CLAUDE.md, এজেন্ট কনফিগ, দক্ষতা, পরিকল্পনা) থাকে।

পুনরায় চালানো পূর্ববর্তী প্যাকেজ প্রতিস্থাপন করে; রেপোতে অন্য কিছু স্পর্শ করা হয় না।

## পুনরুদ্ধার সতর্কতা

`handoff restore` / `unpack` প্রতিটি স্ন্যাপশটকে অবিশ্বস্ত ইনপুট হিসাবে বিবেচনা করে:

- সম্পূর্ণ চেকসাম যাচাইকরণ এবং তালিকা ক্রস-চেক প্রথম;
- পাথ-নিরাপত্তা পরিকল্পনা: জিপ-স্লিপ (`..`), পূর্ণ পথ, ড্রাইভ অক্ষর, অ-`.agentmod` লক্ষ্য, সুরক্ষিত নাম (`.git`, `.ssh`, `.aws`, `.docker`), এবং পালানোর বা পূর্ণ সিম্লিংক লক্ষ্য সবই কিছু লেখার আগে অস্বীকার করা হয়;
- বিদ্যমান `.agentmod/` কে `.agentmod.backup-<stamp>` এ পুনরায় নামকরণ করা হয় আহরণের আগে; যেকোনো ব্যর্থতা স্বয়ংক্রিয়ভাবে এটিতে রোল ব্যাক করে;
- **কখনও কোনো স্ন্যাপশট কার্যকর করা হয় না**;
- তার পরে: Claude গার্ড হুক পুনরায় যুক্ত হয় *এই* মেশিনের বাইনারিতে, মেশিন-নির্দিষ্ট পরম পথগুলি পুনরুদ্ধার করা এজেন্ট কনফিগে সতর্ক করা হয় (আপনার ফাইলগুলি কখনও পুনরায় লেখা হয় না), `doctor` ইনলাইন চলে, এবং প্রয়োজনীয় পুনরায় লগইন পদক্ষেপগুলি মুদ্রিত হয় (প্রমাণপত্র কখনও ভ্রমণ করে না)।

পুনরুদ্ধার অনুমান করার পরিবর্তে অস্বীকার করে — একটি অস্বীকৃত পুনরুদ্ধার প্রজেক্টটি বাইট-অভিন্ন রেখে যায়।

## `agentmod doctor`

শুধুমাত্র পড়া নির্ণয়, যেকোনো সময় চালানোর জন্য নিরাপদ (0 পরিষ্কার, 3 ফলাফল সহ প্রস্থান করুন): প্রজেক্ট/কনফিগ/লেআউট অবস্থা, শেল-হুক ইনস্টলেশন এবং জীবন্ততা, রুটিং ড্রিফ্ট, প্রজেক্টের বাইরে অবশিষ্ট ভেরিয়েবল, ডুপ্লিকেট PATH এন্ট্রি, HOME/শিম লঙ্ঘন, প্রতিটি-এজেন্ট প্রমাণীকরণ উপস্থিতি পুনরায় লগইন নির্দেশনা সহ, OpenCode ফাঁস সতর্কতা, gstack গ্লোবাল/প্রজেক্ট অবস্থা, Claude গার্ড ওয়্যারিং, পুনরুদ্ধার করা কনফিগে বহনযোগ্যতা ঝুঁকি, বিদ্যমান স্ন্যাপশটে রেকর্ড করা গোপনীয় প্রার্থী, `.agentmod-handoff/` এর ভিতরে সেশন/লগ উপাদান, এবং সংগ্রহস্থলের HEAD এখনও সর্বশেষ স্ন্যাপশটের সাথে ম্যাচ করে কিনা।

## Claude Bash গার্ড

`agentmod init` `agentmod guard claude-bash` একটি Claude Code PreToolUse হুক হিসাবে প্রজেক্ট-স্থানীয় হোমে রেজিস্টার করে। এটি Bash কমান্ডগুলি ব্লক করে যা বৈশ্বিক এজেন্ট হোমগুলিতে লিখবে (`~/.claude`, `~/.codex`, `~/.config/opencode`, `~/.local/share/opencode`), `sudo` ব্যবহার করুন, বা `HOME` পুনর্নির্ধারণ করুন — এজেন্ট কারণটি ফিরে পায় এবং সামঞ্জস্য করতে পারে। পড়া কখনও ব্লক করা হয় না। এটি একটি শেল-পার্স হিউরিস্টিক গভীর: দরকারী গার্ডরেইল, স্যান্ডবক্স নয়।

## পরিচিত সীমাবদ্ধতা

সততা বিভাগ। এগুলি অন্তর্নিহিত সরঞ্জামের বৈশিষ্ট্য বা ইচ্ছাকৃত MVP সুযোগ — `doctor` এবং উত্পন্ন ডক্স তাদের সমর্থন করে।

- **macOS Keychain (Claude).** macOS এ Claude Code OAuth প্রমাণপত্রগুলি Keychain এ সংরক্ষণ করে, সমস্ত কনফিগ ডিরে ভাগ করা। প্রতিটি প্রজেক্ট অ্যাকাউন্ট বিচ্ছিন্নতা macOS এ অসম্ভব — এবং প্রজেক্ট প্রতি কোনো পুনরায় লগইনের প্রয়োজন নেই। Linux/Windows প্রতিটি-হোম `.credentials.json` ব্যবহার করে, যা বিচ্ছিন্ন করে কিন্তু প্রতিটি প্রজেক্টে লগইন/অনুলিপি প্রয়োজন।
- **OpenCode আংশিকভাবে ডিফল্টরূপে বিচ্ছিন্ন।** OpenCode এর কোনো একক হোম ভেরিয়েবল নেই; এর কনফিগ একটি মার্জ চেইন যা এখনও বৈশ্বিক `~/.config/opencode/opencode.json` পড়ে, এবং সেশন/সংরক্ষণ/প্রমাণীকরণ বৈশ্বিক XDG ডেটা ডিরে বাস করে। `opencode.xdg_full_isolation = true` সম্পূর্ণ বিচ্ছিন্নতার জন্য XDG ভেরিয়েবলগুলি রুট করে — কিন্তু যা প্রতিটি XDG-সচেতন সরঞ্জামকে প্রভাবিত করে আপনি প্রজেক্টের ভিতরে চালান। `doctor` উভয় পরিস্থিতি রিপোর্ট করে।
- **প্রজেক্ট `.claude/` নেটিভ Claude আচরণ।** Claude Code সর্বদা `./.claude/` পড়ে নির্বিশেষে `CLAUDE_CONFIG_DIR`। agentmod এর Claude এর জন্য যোগ করা মূল্য বিচ্ছিন্ন *ব্যবহারকারী-স্তরের* অবস্থা (বৈশ্বিক দক্ষতা/প্লাগইন, সেশন, ইতিহাস); প্রজেক্ট `.claude/` agentmod এর আগে ইতিমধ্যে কাজ করেছে।
- **প্রথম-সেশন হুক সক্রিয়করণ।** `agentmod init` এর ঠিক পরে, ইতিমধ্যে-চলমান শেল নতুন rc ব্লক লোড করেনি। একটি নতুন টার্মিনাল খুলুন, `exec $SHELL`, অথবা একক-শট `eval "$(agentmod hook zsh)"` (init এটি ঠিক প্রিন্ট করে)। একইভাবে, bash হুক `PROMPT_COMMAND` এর মাধ্যমে ফায়ার করে এবং তাই অ-ইন্টারেক্টিভ bash স্ক্রিপ্টে নিষ্ক্রিয় (direnv এর মতো সীমাবদ্ধতার একই শ্রেণী) — স্ক্রিপ্টগুলি স্পষ্টভাবে ভেরিয়েবলগুলি নির্ধারণ করা উচিত `eval "$(agentmod env --shell bash --activate <root>)"` এর মাধ্যমে যদি তাদের প্রয়োজন হয় রুটিং।
- **শুধুমাত্র npm এর গ্লোবাল বিন PATH এ আছে।** `.agentmod/node/bin` একক পরিচালিত PATH এন্ট্রি। pnpm/bun বৈশ্বিক ইনস্টলগুলি প্রজেক্টে রুট করা হয় (`PNPM_HOME`, `BUN_INSTALL`) কিন্তু তাদের বিন ডিরগুলি PATH এ যুক্ত করা হয় না।
- **ট্রি প্যাকেজগুলি ম্যানুয়ালি পুনরুদ্ধার করে।** `handoff restore` শুধুমাত্র `.amod` ফাইল গ্রহণ করে; একটি প্রতিশ্রুত `.agentmod-handoff/` ডিরেক্টরি এটির ভিতরে `RESTORE.md` অনুসরণ করে পুনরুদ্ধার করা হয় (এই সংস্করণে কোনো ডিরেক্টরি পাঠক নেই)।
- **স্ন্যাপশটগুলি পুনরুদ্ধার-পরে মেরামত প্রয়োজন হতে পারে।** gstack ক্লোন `.git` ছাড়াই ভ্রমণ করে (এটি আবার আপডেটযোগ্য করতে `agentmod install gstack --force` পুনরায় চালান), এবং `node/bin` লঞ্চার সিম্লিংক ঝুলন্ত কারণ `node_modules` বাদ দেওয়া হয় (প্রজেক্টের ভিতরে `npm install -g …` পুনরায় চালান)।
- **শেল সহায়তা zsh এবং bash।** অন্যান্য শেলগুলি এখনও `agentmod env` ম্যানুয়ালি ব্যবহার করতে পারে।

## প্রায়শই জিজ্ঞাসিত প্রশ্ন

**আমি কি সরাসরি `claude` / `codex` / `opencode` ব্যবহার করে থাকি?**
হ্যাঁ। এটাই পয়েন্ট — কোনো মোড়ক, কোনো শিম, কোনো `agentmod run` নেই।

**agentmod কেন শুধু `HOME` পরিবর্তন করে না?**
`HOME` পুনর্নির্ধারণ SSH, git, keychains, dotfiles, এবং শেলে অন্য প্রতিটি সরঞ্জাম ভাঙে। agentmod শুধুমাত্র এজেন্ট-নির্দিষ্ট ভেরিয়েবল রুট করে।

**একটি পুনরুদ্ধার পরে আমার প্রমাণীকরণ কেন অনুপস্থিত?**
ডিজাইন দ্বারা — প্রমাণপত্র কখনও স্ন্যাপশটে ভ্রমণ করে না। নতুন মেশিনে মুদ্রিত পুনরায় লগইন লাইন অনুসরণ করুন (অথবা init এর অনুলিপি অফার)।

**আমি কি `.agentmod/` git এ প্রতিশ্রুত করতে পারি?**
না — init এটি gitignores (সেশন, ক্যাশ, এবং সম্ভবত অনুলিপি করা প্রমাণীকরণ সেখানে বাস করে)। পরিবর্তে নিরাপদ সাবসেট প্রতিশ্রুত করুন: `agentmod pack --for-git`।

**এটি direnv থেকে কীভাবে আলাদা?**
একই সক্রিয়করণ মডেল (ডিরেক্টরি-স্কোপ env, প্রম্পট-হুক ভিত্তিক, প্রস্থানে নিখুঁত পুনরুদ্ধার), কিন্তু agentmod এও জানে *কী* প্রতিটি এজেন্টের জন্য রুট করতে হবে, হোমগুলি তৈরি করে, বৈশ্বিক লেখার বিরুদ্ধে রক্ষা করে, এবং হস্তান্তর করে। দুটি ভাল সহাবস্থান করে।

**একটি স্ন্যাপশট "secret-candidate findings" সহ তৈরি ব্যর্থ।**
সামগ্রী স্ক্যান একটি রাখা ফাইলে ব্যক্তিগত-কী উপাদান পেয়েছে। এটি অপসারণ করুন (অথবা এটি একটি বাদ দেওয়া অবস্থানে সরান যেমন `.env`), অথবা `--allow-findings` সহ যাই হোক প্যাক করুন যদি আপনি এটি স্ন্যাপশটের ভিতরে গ্রহণ করেন।

**এটি Windows এ কাজ করে?**
Go কোড বিল্ড করে এবং Windows-শৈলী পাথের জন্য পাথ সুরক্ষা কার্যকর করা হয়, কিন্তু শেল হুক zsh/bash লক্ষ্য করে; Windows এই সংস্করণে পরীক্ষিত নয়।
