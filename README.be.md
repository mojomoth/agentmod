Ізаляцыя па праектах і перадача для кодавальных агентаў.

`agentmod` дзържыць канфігурацыю, умельі, плагіны, сеансы, кэшы і
рабочы кантэкст **Claude Code**, **Codex CLI** і **OpenCode** унутры
праекта, над якім вы працуеце — і упакоўвае гэтае асяроддзе ў снімак
(snapshot), який можна перадаць іншавай машыне.

Ён падзяляецца на дзве ролі:

1. **Agent Home Router.** Унутры дырэкторыі, якая ўтрымлівае
   `.agentmod/agentmod.toml`, хук абалонкі маршрутызуе дом кожнага агента ў
   `.agentmod/`. Вонкі кожная зменная віднаўляецца ровна так, як яна была, і
   ваша глабальная наладка не датыкаецца.
2. **Handoff Tool.** `agentmod handoff create` упакоўвае `.agentmod/` ў
   праверны `.amod` снімак (або, з `--for-git`, упакаваны дырэкторый файл
   пад `.agentmod-handoff/`). **Git прыносіць ваш код; agentmod прыносіць
   асяроддзе агента.**

## Чым не з'яўляецца agentmod

- **Не пясочніца Docker.** Ён маршрутызуе зменныя асяроддзя ў вашай
  уласнай абалонцы. Няма кантэйнера, няма віртуальнай машыны, няма
  фільтравання syscall.
- **Не поўная ізаляцыя бяспекі.** Інструмент, які ігнорує маршрутызаваныя
  зменныя, можа да сіх пор дасягаць вашых глабальных домаў. Охранная
  система Claude Bash (ніжэй) - гэта глыбокая абарона, а не граніца
  бяспекі.
- **Не ўцечніца.** Ён ніколі не перахоплівае і не загортвае каманды
  `claude`, `codex` ці `opencode`. Вы працягваеце запускаць іх прама,
  без змяненняў.
- **Не інструмент для змены HOME.** `HOME` ніколі не перапрызначаецца.
- **Не інструмент для рэзервного капіявання кода.** Снімкі па дзефолту
  ніколі не ўтрымліваюць ваш зыходны код. Выкарыстоўвайце git для кода.

## Як гэта работает

`agentmod hook zsh` / `agentmod hook bash` выводяць невялічку
самастойную функцыю абалонкі (усталяваную ў ваш файл rc праз `agentmod init`).
На кожным запросе і пры змене дырэкторыі яна ідзе нагору ў пошуках
`.agentmod/agentmod.toml`:

- **Уваход у праект** захоўвае бягучыя значэнні і ўсталяе:

  | Зменная | Маршрутызавана да |
  |---|---|
  | `CLAUDE_CONFIG_DIR` | `.agentmod/claude` |
  | `CODEX_HOME` | `.agentmod/codex` |
  | `OPENCODE_CONFIG` | `.agentmod/opencode/opencode.json` |
  | `NPM_CONFIG_PREFIX` | `.agentmod/node` |
  | `NPM_CONFIG_CACHE` | `.agentmod/node/npm-cache` |
  | `PNPM_HOME` | `.agentmod/node/pnpm` |
  | `BUN_INSTALL` | `.agentmod/node/bun` |
  | `XDG_CONFIG_HOME` / `XDG_DATA_HOME` / `XDG_CACHE_HOME` / `XDG_STATE_HOME` | `.agentmod/opencode/xdg/…` — **толькі** з `opencode.xdg_full_isolation = true` |

  `PATH` палучае ровна адзін запіс, `.agentmod/node/bin` (глабальны бін
  npm пад маршрутызаваным префіксам). Зменныя ўчэту (`AGENTMOD_ACTIVE`,
  `AGENTMOD_PROJECT_ROOT`, `AGENTMOD_ROOT`, `AGENTMOD_VARS`,
  `AGENTMOD_SAVED_*`) фіксуюць, што адмяніць.

- **Выход з праекта** віднаўляе кожнае захаванае значэнне і выдаляе запіс
  `PATH` — ідэальны адварот. Прамое пераключэнне паміж двума праектамі
  agentmod перамаршрутызуе ў адзін крок без ўцечкі ніякіх шляхаў.

Маршрутызацыю на агента можна адключыць у `agentmod.toml`
(`claude.enabled`, `codex.enabled`, `opencode.enabled`, `node.enabled`).

## Ўстаноўка

Выберыце тое, што атрымаецца вам - кожны ўстанаўлівае адзін двайковы
файл:

```sh
# npm (усталяе прарабіны двайковы для вашай платформы)
npm install -g agentmod

# скрыпт усталёўкі (загружае адпаведнае выпуск, праверыць sha256)
curl -fsSL https://raw.githubusercontent.com/mojomoth/agentmod/main/install.sh | sh

# go install (патрабуецца набор інструментаў Go)
go install github.com/mojomoth/agentmod@latest
```

Або соберыце з зыходнага кода (Go 1.26+, адзіны залежнасць модуля
`BurntSushi/toml`):

```sh
git clone https://github.com/mojomoth/agentmod && cd agentmod
go build -o agentmod .
# пастаўце двайковы файл дзе-небудзь на ваш PATH
```

## Хутка пачуцца

```sh
cd ~/work/myproject
agentmod init          # стварае .agentmod/, рэдагуе .gitignore, усталяе
                       # хук абалонкі ў файл rc, прапаноўвае скапіяваць auth
# толькі першы раз: хук яшчэ не дзяйсны ў ГЭТАЙ абалонцы —
# адкрыйце новы тэрмінал, ці: exec $SHELL

cd ~/work/myproject    # хук актывуецца; праверыць:
agentmod status        # "AgentMod: active", маршрутызаваныя шляхі лічацца
claude                 # простая каманда — цяпер выкарыстоўвае мясцовы дом праекта
agentmod install gstack   # мясцовыя умельі, глабальны дом не датыкаецца

agentmod pack          # снімак да .agentmod/snapshots/<name>-<stamp>.amod
agentmod doctor        # дыягностыка толькі для чытання ў любы час
```

На машыне, якая атрымлівае:

```sh
cd ~/work/myproject    # зыходнік прыбыў праз git
agentmod init
agentmod unpack myproject-20260611-123045.amod
# ідзіце за надрукаванымі нотамі рэ-логіна; doctor запускаецца аўтаматычна
```

## `agentmod init`

Ідэмпатэнтны — перазапуск запаўняе што-небудзь, што не хватае, і ніколі не
перапісвае існуючы `agentmod.toml` ці ніякі файл карыстальніка. Ён:

- стварае `.agentmod/{claude,codex,opencode,node,snapshots,logs}` і
  дзефолтны `agentmod.toml`;
- прывязвае охранную сістему Claude Bash да `.agentmod/claude/settings.json`;
- дадае `.agentmod/` да `.gitignore` (створаны толькі ў рэпазіторыі git);
- усталяе хук абалонкі як агароджаны блок у `~/.zshrc` ці `~/.bashrc`
  (ваша абалонка з `$SHELL`; блок абнаўляецца на месцы, ніколі не
  дублюецца, і ваш асобны змест rc не датыкаецца);
- прапаноўвае **скапіяваць** існуючыя файлы auth Claude/Codex ў мясцовы
  дом праекта (гл. "Auth" ніжэй) — копіяванне адбываецца толькі на
  відавочны `y`.

Флаги: `--no-shell-hook` пропускае ўсе рэдакцыі файла rc;
`--yes` / `--non-interactive` ніколі не пытаецца і таму ніколі не
копіюе auth (для CI).

## Выкарыстанне звычайных `claude`, `codex`, `opencode`

Няма каманды-ўцечніцы. Унутры актыўнага праекта звычайныя каманды проста
бачаць маршрутызаваныя домы:

- **Claude Code** чытае `CLAUDE_CONFIG_DIR` → мясцовыя наладкі праекта,
  умельі карыстальніка/плагіны, сеансы, гісторыя. (Мясцовы `.claude/`
  **адвека** чытаецца натуральна — гл. Абмежаванні.)
- **Codex CLI** чытае `CODEX_HOME` → мясцовы `config.toml`, `auth.json`,
  сеансы, гісторыя, журналы.
- **OpenCode** чытае `OPENCODE_CONFIG` → мясцовы файл канфігурацыі. Гэта
  **частковая** ізаляцыя па дзефолту — гл. Абмежаванні.

### Auth

Свежыя мясцовыя домы пачынаюцца без ўдоставеранняў:

- **Claude на macOS**: нічога не трэба рабіць — ўдоставеранні жывуць у
  Keychain і дзяляюцца з кожным дырэкторыем канфігурацыі (гэта таксама
  значыць, што яны **не** ізаліраны па праектах).
- **Claude на Linux/Windows**: запусціце `claude login` унутры праекта,
  або прыміце прапанову init скапіяваць `~/.claude/.credentials.json`.
- **Codex**: запусціце `codex login` унутры праекта, або прыміце
  прапанову init скапіяваць `~/.codex/auth.json`.

Auth файлы **ніколі не падарожнічаюць у снімках** (выключаны па імені,
незалежна ад таго, як яны атрымаліся там).

## Усталяванне gstack

[gstack](https://github.com/garrytan/gstack) захардкодвае свойны
усталяльнік да `~/.claude/skills/gstack` — ровна глабальнае забруджанне,
якое agentmod існуе для пабілюцы. Так:

```sh
agentmod install gstack            # клон у .agentmod/claude/skills/gstack
agentmod install gstack --force    # замяніць існуючы мясцовы ўстаноўку
```

Усталяльнік клонуе з git, ніколі не запускае свойны скрыпт усталёўкі gstack, і
робіць снімак спіса `~/.claude/skills` да і пасля — любая змена ў
глабальным дырэкторыі паведамляецца як парушэнне і не адбываецца.
`agentmod doctor` асобна папярэджвае кожны раз, калі існуе **глабальны**
ўстаноў gstack (нават адзін, які вы ўстанавілі самі да таго, як прыняўся
agentmod), таму што глабальна ўсталяваныя умельі цяўчуць у кожны праект.

## Handoff (`.amod` снімкі)

```sh
agentmod handoff create [--output PATH] [--allow-findings] [--allow-dirty]
agentmod handoff list
agentmod handoff inspect FILE      # маніфест + рапорт рэдакцыі, без экстракцыі
agentmod handoff verify  FILE      # перахешаваць кожнага члена; выхад 3 пры невяданні
agentmod handoff restore FILE      # замяніць .agentmod/ (рэзервная копія праведзена)
agentmod pack / agentmod unpack    # аліасы create / restore
```

Снімак - гэта zip з шасцьма кораннямі членамі — `manifest.json`,
`inventory.json` (размер/sha256/режым на файл), `REDACTION.md` (што было
выключана і чаму, плюс вывады сканавання сакретаў), `HANDOFF.md` і `RESTORE.md`
(інструкцыі для прымача), `checksums.txt` (`shasum -a 256 -c`-сумяшчальны)
— і нагрузка пад `payload/.agentmod/…`. Стварэнне атамічны і дэтэрмінавана;
маніфест фіксуе ветку git/commit/грудны стан з ніякімі ўдоставеранням,
очищаныя з аддаленага URL. Грудны вёр адхіляе пакет, калі не `--allow-dirty`.

`inspect` і `verify` работаюць везда — прыймач можа аўдытаваць снімак да таго,
як любы праект быў ўстаноўлены.

### Палітыка выключэння сакретаў

Два слоі, обе па дзефолту ўключаны:

1. **Правілы выключэння** выдаляюць вядомыя чувальныя файлы з нагрузкі і
   лічаць кожны адзін у `REDACTION.md`: auth файлы па імені (`.credentials.json`,
   `auth.json`, `credentials*`), `*.env` / `.env.*`, SSH ключы (`id_*`,
   `*.pem`, `*.pub`), дырэкторыі ўдоставерання (`.ssh`, `.aws`, `.azure`,
   `.gcloud`, `.kube`, `.gnupg`, `.docker`), файлы keychain, `.git`,
   `node_modules`, кэшы і тэмпавы дырэкторыі.
2. **Скан змісцу** па кожнаму **захаванаму** файлу. Матэрыял прыватнага
   ключа адхіляе стварэнне зусім, калі не прайсце `--allow-findings` (і
   адзначаны як HARD у `REDACTION.md`). Вероятныя токены (AWS access key IDs,
   GitHub токены, `sk-…` ключы, `api_key=`-стыльныя прысвойванні)
   папярэджваюцца, але не блокіруюцца.

Скан - гэта эўрыстыка. **Прагледжыце `REDACTION.md` (ці `handoff inspect`)
перад агульнай поділкай снімка** — сеансы і рабочы кантэкст падарожнічаюць па
дызайну і могуць цытаваць што-небудзь, што вы злапалі ў агента разважаў.
Снімкі напісаны рэжым 0600 па гэтай прычыне; адносьцеся да іх як да
прыватных файлаў.

## Git handoff

```sh
agentmod pack --for-git    # напішчыке .agentmod-handoff/ у каране праекта
git add .agentmod-handoff && git commit
```

Тыя самыя шэсьцэ членаў і нагрузка, як `.amod`, але як адкрыты дырэкторый
простых файлаў (`shasum -a 256 -c checksums.txt` працует ў дырэкторыі). На
вяршыні дзефолтных выключэнняў ён чышчыць **сеансы, транскрыпцыі,
гісторыю, і журналы** для ўсіх трох агентаў — тыя рэгулярна ўтрымліваюць
пасцічы сакретаў і не адносяцца да рэпазіторыі. `--include-sessions` адвека
адхіляе: захаванне сеансаў патрабавала б шыфравання, якое гэта версія не
рэалізуе. Рабочы кантэкст, які бяспечны для сумесніцтва (CLAUDE.md, канфігі
агента, умельі, планы) захоўваюцца.

Пераскід замяняе папярэдні пакет; нічога ў рэпазіторыі не датыкаецца.

## Меры пры восстанаўленні

`handoff restore` / `unpack` лічыць кожны снімак як ненадзейны ўвод:

- поўная праверка sha256 і инвентарыўв крыж-чэк спярёд;
- план бяспекі шляхаў: zip-slip (`..`), абсалютныя шляхі, лётчыкі ўводу,
  не-`.agentmod` мэты, абаронь імёнаў (`.git`, `.ssh`, `.aws`, `.docker`), і
  экранаванне або абсалютныя мішэні сімвальных спасылак адхіляюцца перад
  тым, як што-небудзь пісаць;
- існуючы `.agentmod/` перапрызначаны да `.agentmod.backup-<stamp>` перад
  экстракцыяй; любы памылка вяртаецца аўтаматычна;
- **нічога з снімка ніколі не выконваецца**;
- пасля: охранная система Claude пераўстанаўлівае за гэтай машыны
  двайковы, машынна-спецыфічныя абсалютныя шляхі, знойдзеныя ў
  васстановелных канфігах агента, папярэджваюцца (ваш файлы ніколі не
  змяняюцца), `doctor` запускаецца автаматычна, і патрэбныя крокі рэ-логіна
  паказаны (auth ніколі не адпраўляецца).

Восстановленні адхіляюць, а не угадваюць — адхілены restore пакідае праект
байтавай ідэнтычным.

## `agentmod doctor`

Дыягностыка толькі для чытання, бяспечна запускаць ў любы час (выход 0
чысты, 3 з вывадамі): праект/канфіг/макет стан, ўстаноўка хука абалонкі і
жывость, дрэйф маршрутызацыі, скарачальныя зменныя вонкі праектаў,
дублёныя записы PATH, HOME/shim парушэнні, прысутнасць auth на агента з
інструкцыямі рэ-логіна, папярэджанні OpenCode цяўчу, gstack глабальны/праект
стан, охранная сістема Claude прывязаны, мёжнасць парушэнняў у
восстановелных канфіг, сакрэт кандыдаты запісаны ў існуючых снімках,
матэрыял сеансу/журнал унутры `.agentmod-handoff/`, і ці той ды HEAD
рэпазіторыі адпавядае найнавейшаму снімку.

## Охранная сістема Claude Bash

`agentmod init` рэгіструе `agentmod guard claude-bash` як Claude Code
PreToolUse хук у мясцовы дом праекта. Ён блокіруе Bash каманды, якія
пішуць да глабальных домаў агента (`~/.claude`, `~/.codex`,
`~/.config/opencode`, `~/.local/share/opencode`), выкарыстоўваюць `sudo`,
або перапрызначаюць `HOME` — агент атрымлівае прычыну назад і можа
адрегуляваць. Чытанні ніколі не блокіруюцца. Гэта адзін паўн-разбор
эўрыстыка: карысны парапет, а не пясочніца.

## Вядомыя абмежаванні

Раздзел чесности. Гэта ўласцівасцы базавых інструментаў ці натуральны
адбор MVP — `doctor` і генерыраваная дакументацыя распавядаюць аб іх таксама.

- **macOS Keychain (Claude).** Claude Code на macOS захоўвае ўдоставеранні
  OAuth ў Keychain, дзяліцца па **ўсіх** дырэкторыях канфіг. Ізаляцыя
  алікунту па праектах немажлива на macOS — і ніякі рэ-логін патрабуецца
  па праектах. Linux/Windows выкарыстоўваюць па-дома `.credentials.json`,
  якая ізалюе, але патрабуе логіна/копіі па праектах.
- **OpenCode часткова ізаліруецца па дзефолту.** OpenCode няма адзіны
  домаў зменнай; яго канфіг - гэта заліўка ланцуга, якая да сіх пор
  чытае глабальны `~/.config/opencode/opencode.json`, і сеансы/сховіша/auth
  жывуць у глабальных дырэкторыях XDG. `opencode.xdg_full_isolation = true`
  маршрутызуе зменныя XDG для поўнай ізаляцыі — але гэта датыкаецца
  **кожнага** інструмента XDG-свядомага, які вы запускаеце унутры праекта.
  `doctor` паведамляе абедзве сітуацыі.
- **Мясцовы `.claude/` - натуральны Claude павёданне.** Claude Code адвека
  чытае `./.claude/` незалежна ад `CLAUDE_CONFIG_DIR`. Даданая вартасць
  agentmod для Claude ізалюе **карыстальніка-ўзроўня** стан (глабальныя
  умельі/плагіны, сеансы, гісторыя); мясцовы `.claude/` ужо работав да
  agentmod.
- **Першы-сеанс хук актывацыя.** Прама пасля `agentmod init`, ужо
  запусціны абалонка не загрузіў новы блок rc. Адкрыйце новы тэрмінал,
  `exec $SHELL`, ці адзін-шот `eval "$(agentmod hook zsh)"` (init
  вывядзе ровна гэта). Аналагічна, bash хук прыходзіцца праз `PROMPT_COMMAND`
  і таму неактыўны ў неінтэрактыўных bash скрыптах (тыя ж клас абмежавання
  як direnv) — скрыпты павінны ўсталяваць зменныя відавочна праз
  `eval "$(agentmod env --shell bash --activate <root>)"` калі яны
  патрабуюць маршрутызацыі.
- **Толькі глабальны бін npm на PATH.** `.agentmod/node/bin` - гэта адзіны
  кіруемы запіс PATH. Глабальныя ўсталёўкі pnpm/bun маршрутызуюцца ў
  праект (`PNPM_HOME`, `BUN_INSTALL`), але іх дырэкторыі бінаў не дадаюцца да
  PATH.
- **Дрэвавыя пакеты васстанаўліваюцца ручнога.** `handoff restore` прынімае
  файлы `.amod` толькі; камітны `.agentmod-handoff/` дырэкторый
  васстанаўляецца па інструкцыяў з `RESTORE.md` унутры яго (гэта версія не
  можа чытаць дырэкторыі).
- **Снімкі могуць патрабаваць пасля-восстанаўлення рамонту.** Клон gstack
  падарожнічае без яго `.git` (пераскід `agentmod install gstack --force`,
  каб зрабіць яго абновлены зноў), і `node/bin` запускаў сімвальныя спасылкі
  памяцець, таму што `node_modules` выключаны (пераскід `npm install -g …`
  унутры праекта).
- **Абалонка падтрымкі - гэта zsh і bash.** Іншыя абалонкі могуць да сіх
  пор выкарыстоўваць `agentmod env` ручнога.

## FAQ

**Ці трымаю я выкарыстанне `claude` / `codex` / `opencode` прамо?**
Так. Гэта пункт — няма ўцечніц, няма шімаў, няма `agentmod run`.

**Чаму agentmod просто не змяніць `HOME`?**
Перапрызначэнне `HOME` разбівае SSH, git, keychains, dotfiles, і кожны
інш інструмент у абалонцы. agentmod маршрутызуе толькі зменныя,
спецыфічныя для агента.

**Чаму мой auth прапаў пасля васстанаўлення?**
По дызайну — ўдоставеранні ніколі не адпраўляюцца у снімках. Следуйце
надрукаванымі лініямі рэ-логіна (ці прыміце прапанову ініцыяла) на новай машыне.

**Ці магу я зацвёрдзіць `.agentmod/` да git?**
Няет — init gitignores яго (сеансы, кэшы, і магчама скапіяваны auth жывуць
там). Зацвёрдзіце бяспечны падмноства замест: `agentmod pack --for-git`.

**Як гэта адрозніваецца ад direnv?**
Адна мадэль актывацыі (дырэкторый-абмежаваны асяроддзе, хук-запроса базаваны,
ідэальны восстановіць на выхад), але agentmod таксама ведае **што** маршрутызаваць
для кожнага агента, створыць домы, охранніць ад глабальных пісанняў, і
робіць handoff. Яны адлічна дзяйнічаюць вместе.

**Снімак не атрымаецца стварыць з "secret-candidate вывадамі".**
Скан змісцу знойшаў матэрыял прыватнага ключа ў захаванаму файлу. Выдаліце яго
(ці перадвіжьце яго да выключанага месца, як `.env`), ці пакуйце ў любім
выпадку з `--allow-findings`, калі вы прынімаеце яго быць унутры снімка.

**Ці працует гэта на Windows?**
Код Go збірацца і бяспека шляхаў толькі для Windows-стыльных шляхаў, але
хукі абалонкі рассчітаны на zsh/bash; Windows ніколі не тэставалась у гэтай версіі.
