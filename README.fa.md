انزوا و تحویل پروژه هر کدام برای عوامل کدنویسی.

`agentmod` پیکربندی، مهارت‌ها، افزونه‌ها، جلسات، حافظه پنهان و متن کار **Claude Code**، **Codex CLI** و **OpenCode** را درون پروژه‌ای که روی آن کار می‌کنید نگه می‌دارد — و آن محیط را به یک تصویر منطبق می‌کند که می‌توانید به ماشینی دیگر تحویل دهید.

این دو نقش ایفا می‌کند:

1. **عامل Home Router.** درون یک درختواره دایرکتوری که شامل
   `.agentmod/agentmod.toml` است، یک هوک shell خانه هر عامل را به
   `.agentmod/` مسیریابی می‌کند. خارج از آن، هر متغیری دقیقاً همانطور که بود بازگردانده می‌شود و
   تنظیم جهانی شما دست نخورده است.
2. **ابزار تحویل.** `agentmod handoff create` یک فایل
   `.amod` منطبق قابل تأیید را بسته‌بندی می‌کند (یا با `--for-git`، یک درخت فایل قابل عهدنامه
   تحت `.agentmod-handoff/`). **Git منبع شما را منتقل می‌کند؛ agentmod محیط عامل را منتقل می‌کند.**

## agentmod چیست *نیست*

- **داخل یک صندوق Docker نیست.** متغیرهای محیط را در پوسته خود مسیریابی می‌کند.
  هیچ کانتینری، هیچ ماشین مجازی، هیچ فیلتر syscall وجود ندارد.
- **انزوای امنیت کامل نیست.** ابزاری که متغیرهای مسیریابی شده را نادیده می‌گیرد
  همچنان می‌تواند به خانه‌های جهانی دسترسی پیدا کند. حفاظ Bash Claude (زیر) تدافع عمیق است، نه مرز امنیتی.
- **یک shim نیست.** هرگز دستورات `claude`، `codex` یا
  `opencode` را قطع یا بسته‌بندی نمی‌کند. شما به اجرای آن‌ها به طور مستقیم، بدون تغییر ادامه می‌دهید.
- **ابزاری که `HOME` را تغییر می‌دهد نیست.** `HOME` هرگز مجدداً اختصاص داده نمی‌شود.
- **ابزار پشتیبان‌گیری کد منبع نیست.** تصاویر منطبق هرگز شامل کد منبع شما به طور پیش‌فرض نیستند. از git برای منبع استفاده کنید.

## چطور کار می‌کند

`agentmod hook zsh` / `agentmod hook bash` یک تابع shell خودکفا کوچک را چاپ می‌کند (توسط `agentmod init` در فایل rc شما نصب شده است). در هر پرامپت و تغییر دایرکتوری، به دنبال `.agentmod/agentmod.toml` حرکت می‌کند:

- **وارد شدن به یک پروژه** مقادیر فعلی را ذخیره می‌کند و تنظیم می‌کند:

  | متغیر | مسیریابی شده به |
  |---|---|
  | `CLAUDE_CONFIG_DIR` | `.agentmod/claude` |
  | `CODEX_HOME` | `.agentmod/codex` |
  | `OPENCODE_CONFIG` | `.agentmod/opencode/opencode.json` |
  | `NPM_CONFIG_PREFIX` | `.agentmod/node` |
  | `NPM_CONFIG_CACHE` | `.agentmod/node/npm-cache` |
  | `PNPM_HOME` | `.agentmod/node/pnpm` |
  | `BUN_INSTALL` | `.agentmod/node/bun` |
  | `XDG_CONFIG_HOME` / `XDG_DATA_HOME` / `XDG_CACHE_HOME` / `XDG_STATE_HOME` | `.agentmod/opencode/xdg/…` — **فقط** با `opencode.xdg_full_isolation = true` |

  `PATH` دقیقاً یک ورودی به دست می‌آورد، `.agentmod/node/bin` (bin جهانی npm
  تحت پیشوند مسیریابی شده). متغیرهای حسابرسی (`AGENTMOD_ACTIVE`,
  `AGENTMOD_PROJECT_ROOT`, `AGENTMOD_ROOT`, `AGENTMOD_VARS`,
  `AGENTMOD_SAVED_*`) آنچه را ثبت می‌کنند که باید واگردانده شود.

- **ترک کردن پروژه** هر مقدار ذخیره شده را بازگردانده و ورودی `PATH` را حذف می‌کند — یک معکوس کامل. تبدیل مستقیم بین دو پروژه agentmod در یک گام بدون نشت از هر دوی مسیرهای پروژه مجدداً مسیریابی می‌کند.

مسیریابی به ازای هر عامل را می‌توان در `agentmod.toml` خاموش کرد
(`claude.enabled`, `codex.enabled`, `opencode.enabled`, `node.enabled`).

## نصب

هر کدام را انتخاب کنید که با تنظیمات شما مطابقت دارد — هر کدام یک باینری منفرد را نصب می‌کند:

```sh
# npm (نصب باینری از پیش‌ساخت شده برای پلتفرم شما)
npm install -g agentmod

# اسکریپت نصب (انتشار مطابق را دانلود می‌کند، sha256 را تأیید می‌کند)
curl -fsSL https://raw.githubusercontent.com/mojomoth/agentmod/main/install.sh | sh

# go install (نیاز به زنجیره ابزار Go دارد)
go install github.com/mojomoth/agentmod@latest
```

یا از منبع ساخت کنید (Go 1.26+، تنها وابستگی ماژول
`BurntSushi/toml`):

```sh
git clone https://github.com/mojomoth/agentmod && cd agentmod
go build -o agentmod .
# باینری را جایی در PATH شما قرار دهید
```

## شروع سریع

```sh
cd ~/work/myproject
agentmod init          # .agentmod/ را ایجاد می‌کند، .gitignore را ویرایش می‌کند، نصب می‌کند
                       # shell hook در فایل rc شما، تجویز کپی کردن احراز هویت
# فقط بار اول: هوک در این پوسته نصب نشده است —
# یک ترمینال جدید باز کنید، یا: exec $SHELL

cd ~/work/myproject    # هوک فعال می‌شود؛ آن را بررسی کنید:
agentmod status        # "AgentMod: active"، مسیرهای مسیریابی شده فهرست شده‌اند
claude                 # دستور ساده — اکنون از خانه محلی پروژه استفاده می‌کنید
agentmod install gstack   # مهارت‌های محلی پروژه، خانه جهانی دست نخورده است

agentmod pack          # تصویر منطبق برای .agentmod/snapshots/<name>-<stamp>.amod
agentmod doctor        # تشخیص بدون نوشتار، هر زمان
```

در ماشین دریافت‌کننده:

```sh
cd ~/work/myproject    # منبع از طریق git رسیده است
agentmod init
agentmod unpack myproject-20260611-123045.amod
# از یادداشت‌های دوباره ورود چاپ شده پیروی کنید؛ doctor خودکار اجرا می‌شود
```

## `agentmod init`

Idempotent — دوباره اجرا هر چیزی را که غایب است پر می‌کند و هیچگاه
`agentmod.toml` موجود یا فایل کاربر را بازنویسی نمی‌کند. این کار انجام می‌دهد:

- `.agentmod/{claude,codex,opencode,node,snapshots,logs}` و یک `agentmod.toml` پیش‌فرض را ایجاد می‌کند؛
- حفاظ Bash Claude را در `.agentmod/claude/settings.json` متصل می‌کند؛
- `.agentmod/` را به `.gitignore` اضافه می‌کند (فقط درون یک مخزن git ایجاد شد)؛
- هوک shell را به عنوان یک بلوک محدود شده در `~/.zshrc` یا `~/.bashrc` نصب می‌کند
  (شما پوسته از `$SHELL`؛ بلوک در جای خود بهنگام می‌شود، هرگز
  تکرار نشده‌است، و محتوای rc شما هرگز لمس نشده‌است)؛
- تجویز **کپی کردن** فایلهای احراز هویت Claude/Codex موجود به محل خانه محلی پروژه (بخش "Auth" را ببینید) — کپی کردن تنها در یک `y` صریح اتفاق می‌افتد.

پرچم‌ها: `--no-shell-hook` همه ویرایش‌های فایل rc را نادیده می‌گیرد؛ `--yes` /
`--non-interactive` هرگز پرامپت نمی‌کند و بنابراین احراز هویت را کپی نمی‌کند (برای CI).

## استفاده از `claude`، `codex`، `opencode` ساده

هیچ دستور بسته‌بندی وجود ندارد. درون یک پروژه فعال دستورات عادی ساده‌اً خانه‌های مسیریابی شده را می‌بینند:

- **Claude Code** `CLAUDE_CONFIG_DIR` → تنظیمات محلی پروژه را می‌خواند،
  مهارت‌های سطح کاربر/افزونه‌ها، جلسات، تاریخچه. (پروژه `.claude/`
  است *همیشه* به طور بومی خوانده می‌شود — بخش محدودیت‌ها را ببینید.)
- **Codex CLI** `CODEX_HOME` → پروژه-محلی `config.toml` را می‌خواند،
  `auth.json`، جلسات، تاریخچه، سیاق‌نویسی.
- **OpenCode** `OPENCODE_CONFIG` → فایل پیکربندی محلی پروژه را می‌خواند.
  این انزوای *جزئی* به طور پیش‌فرض است — بخش محدودیت‌ها را ببینید.

### احراز هویت

خانه‌های محلی پروژه تازه‌ای بدون اعتبار شروع می‌شوند:

- **Claude در macOS**: چیزی انجام دهید — اعتبار در Keychain زندگی می‌کند و
  با هر دایرکتوری پیکربندی مشترک است (که به این معنی است که آن‌ها *نیستند*
  هر پروژه جداگانه منزوی).
- **Claude در Linux/Windows**: `claude login` را درون پروژه اجرا کنید، یا
  تجویز init را قبول کنید `~/.claude/.credentials.json` کپی کنید.
- **Codex**: `codex login` را درون پروژه اجرا کنید، یا تجویز init را قبول کنید
  `~/.codex/auth.json` کپی کنید.

فایلهای احراز هویت **هرگز در تصاویر منطبق سفر نمی‌کنند** (توسط نام مستثنی، صرف‌نظر از
چگونه آن‌ها به آنجا رسیدند).

## نصب gstack

[gstack](https://github.com/garrytan/gstack) نصب‌کننده خود را به
`~/.claude/skills/gstack` — دقیقاً آلودگی جهانی agentmod سخت‌گیری می‌کند
برای جلوگیری. بنابراین:

```sh
agentmod install gstack            # درون .agentmod/claude/skills/gstack کلون می‌کند
agentmod install gstack --force    # یک نصب محلی پروژه موجود را جایگزین می‌کند
```

نصب‌کننده با git کلون می‌کند، هرگز اسکریپت تنظیم خود gstack را اجرا نمی‌کند، و
فهرست `~/.claude/skills` را قبل و بعد از اسنپ‌شات می‌کند — هر تغییری در
دایرکتوری جهانی به عنوان نقض گزارش می‌شود و دستور را شکست می‌دهد.
`agentmod doctor` جداگانه هنگامی که یک نصب gstack *جهانی* موجود است هشدار می‌دهد
(حتی یکی که خود نصب کردید قبل از پذیرش agentmod)، زیرا مهارت‌های نصب شده‌ی جهانی
به هر پروژه نشت می‌کنند.

## تحویل (تصاویر `.amod` منطبق)

```sh
agentmod handoff create [--output PATH] [--allow-findings] [--allow-dirty]
agentmod handoff list
agentmod handoff inspect FILE      # manifest + گزارش redaction، بدون استخراج
agentmod handoff verify  FILE      # هر عضو را دوباره هش کنید؛ در مورد عدم تطابق 3 را خروج کنید
agentmod handoff restore FILE      # .agentmod/ را جایگزین کنید (پشتیبان‌گیری نخست اتخاذ شد)
agentmod pack / agentmod unpack    # نام‌های مستعار ایجاد / بازگردانی
```

یک تصویر منطبق یک زیپ با شش عضو ریشه است — `manifest.json`،
`inventory.json` (سایز/sha256/mode هر فایل)، `REDACTION.md` (آنچه بیرون شد و چرا، به اضافه
یافته‌های اسکن مخفی)، `HANDOFF.md` و `RESTORE.md`
(دستورالعمل‌های انسانی برای گیرنده)، `checksums.txt`
(`shasum -a 256 -c`-سازگار) — و بار کاری تحت
`payload/.agentmod/…`. ایجاد انجام‌شود atomic و قطعی است؛ manifest
شاخه/commit/حالت کثیف git را با هر اعتبار نوار از URL دور ریخته شده ثبت می‌کند. یک
درخت کاری کثیف نشت را رد می‌کند مگر `--allow-dirty`.

`inspect` و `verify` هر جایی کار می‌کنند — گیرنده می‌تواند قبل از داشتن
هر پروژه تنظیم شده یک تصویر منطبق را بررسی کند.

### سیاست مستثنیات مخفی

دو لایه، هر دو به طور پیش‌فرض فعال:

1. **قوانین مستثناء** فایلهای حساس شناخته شده را از بار کاری بیرون می‌اندازند و هر یک را فهرست می‌کنند
   در `REDACTION.md`: فایلهای احراز هویت به نام (`.credentials.json`،
   `auth.json`، `credentials*`)، `*.env` / `.env.*`، کلیدهای SSH (`id_*`،
   `*.pem`، `*.pub`)، دایرکتوری‌های اعتبار (`.ssh`، `.aws`، `.azure`،
   `.gcloud`، `.kube`، `.gnupg`، `.docker`)، فایلهای Keychain، `.git`،
   `node_modules`، حافظه پنهان و دایرکتوری‌های temp.
2. **اسکن محتوای** در هر فایل *نگهداری شده*. ماده کلید خصوصی ایجاد را رد می‌کند
   کاملاً مگر اینکه `--allow-findings` را بگذارید (و سپس HARD در `REDACTION.md` نشانه‌گذاری می‌شود). نشانه‌های احتمالی توکن (AWS access key IDs، GitHub
   توکن‌ها، `sk-…` کلید‌ها، `api_key=`-تخصیص سبک) درباره هشدار داده می‌شوند اما
   مسدود نمی‌شوند.

اسکن heuristic است. **`REDACTION.md` را بررسی کنید (یا `handoff inspect`) قبل از
به اشتراک گذاشتن یک تصویر منطبق** — جلسات و متن کار سفر طرح‌ریزی شده‌اند و ممکن است
هر چیزی را که شما به یک صحبت عامل شامل کردید نقل کنند. تصاویر منطبق با mode 0600 برای این دلیل
نوشته می‌شوند؛ آن‌ها را مانند فایلهای خصوصی رفتار کنید.

## تحویل Git

```sh
agentmod pack --for-git    # `.agentmod-handoff/` را در ریشه پروژه می‌نویسد
git add .agentmod-handoff && git commit
```

همان شش عضو و بار کاری به عنوان یک `.amod`، اما به عنوان یک درخت قابل عهدنامه
فایل‌های ساده (`shasum -a 256 -c checksums.txt` در دایرکتوری کار می‌کند). بر فراز
استثنای پیش‌فرض آن نوار **جلسات، رونوشت‌ها، تاریخچه،
و سیاق‌نویسی‌های** هر سه عامل — آن‌ها معمولاً شامل مخفی‌های شامل شده و
مخزن را متعلق نیستند. متن کار که ایمن برای اشتراک است (CLAUDE.md، پیکربندی عامل،
مهارت‌ها، برنامه‌ریزی) باقی می‌ماند.

دوباره اجرا بسته قبلی را جایگزین می‌کند؛ هیچ چیز دیگر در repo نیست
لمس شده.

## احتیاطات بازگردانی

`handoff restore` / `unpack` هر تصویر منطبق را به عنوان ورودی عدم اعتماد سخت‌گیری می‌کند:

- اولین تأیید checksum و بررسی فهرست متقابل کامل؛
- طرح ایمنی مسیر: zip-slip (`..`)، مسیرهای مطلق، نامه‌های درایو،
  اهداف غیر-`.agentmod`، نام‌های محافظت شده (`.git`، `.ssh`، `.aws`،
  `.docker`)، و فرار از یا اهداف symlink مطلق همه رد می‌شوند
  پیش از نوشتن هر چیز؛
- `.agentmod/` موجود به `.agentmod.backup-<stamp>` قبل از
  استخراج نام‌گذاری مجدد می‌شود؛ هر شکست به طور خودکار به آن بازگردانیده می‌شود؛
- **هیچ چیز از یک تصویر منطبق هرگز اجرا نمی‌شود**؛
- پس از آن: هوک حفاظ Claude دوباره به باینری *این* ماشین متصل می‌شود،
  مسیرهای مطلق خاص ماشین پیدا شده در پیکربندی عامل بازگردانده شده هشدار داده می‌شوند (فایل‌های شما
  هرگز دوباره نوشته نمی‌شوند)، `doctor` درون خطی اجرا می‌شود، و
  مراحل دوباره ورود لازم چاپ می‌شوند (احراز هویت هرگز سفر نمی‌کند).

بازگردانی‌ها رد می‌کنند بجائے حدس — یک بازگردانی رد شده پروژه را
بایت-یکسان برای اندازه گذار می‌کند.

## `agentmod doctor`

تشخیص بدون نوشتار، ایمن برای اجرا هر زمان (exit 0 تمیز، 3 با یافته‌ها):
پروژه/پیکربندی/وضعیت طرح، نصب هوک shell و نرم‌افزار، رانش مسیریابی،
متغیرهای سرگردان خارج پروژه‌ها، ورودی‌های PATH تکراری،
نقض HOME/shim، حضور احراز هویت به ازای هر عامل با دستورالعمل دوباره ورود،
هشدار‌های نشت OpenCode، وضعیت gstack جهانی/پروژه، Claude حفاظ وایرینگ،
خطرات توزیع‌پذیری در پیکربندی‌های بازگردانده شده، نامزدهای مخفی ثبت شده در
تصاویر منطبق موجود، ماده جلسه/سیاق‌نویسی درون `.agentmod-handoff/`، و
اینکه آیا HEAD مخزن هنوز تطابق دارد تازه‌ترین تصویر منطبق را.

## حفاظ Bash Claude

`agentmod init` `agentmod guard claude-bash` را به عنوان Claude Code
PreToolUse هوک درون خانه محلی پروژه ثبت می‌کند. این Bash دستورات را مسدود می‌کند که
به خانه‌های عامل جهانی (`~/.claude`، `~/.codex`،
`~/.config/opencode`، `~/.local/share/opencode`) می‌نویسند، از `sudo` استفاده می‌کنند، یا
`HOME` را مجدداً اختصاص می‌دهند — عامل دلیل را برگردانده و می‌تواند تنظیم کند. خوانش‌ها هرگز
مسدود نمی‌شوند. این یک heuristic تجزیه shell عمیق است: حفاظ مفید، نه یک
sandbox.

## محدودیت‌های شناخته شده

بخش صداقت. این خواص ابزارهای اساسی یا دامنه MVP عمدی‌ای هستند — `doctor` و
اسناد تولید شده آن‌ها را بیان می‌کنند.

- **Keychain macOS (Claude).** Claude Code در macOS اعتبار OAuth را
  در Keychain نگه‌داشته‌اند، مشترک *تمام* دایرکتوری‌های پیکربندی. انزوای حساب هر پروژه
  در macOS غیرممکن است — و دوباره ورود هر پروژه لازم نیست.
  Linux/Windows یک per-home `.credentials.json` استفاده می‌کنند، که منزوی اما
  هر پروژه ورود/کپی لازم می‌کند.
- **OpenCode جزئی منزوی به طور پیش‌فرض است.** OpenCode هیچ متغیر خانه منفرد ندارد؛ پیکربندی آن
  یک زنجیره ادغام است که هنوز جهانی را می‌خواند
  `~/.config/opencode/opencode.json`، و جلسات/ذخیره‌سازی/احراز هویت در
  دایرکتوری‌های XDG جهانی. `opencode.xdg_full_isolation = true` XDG را مسیریابی می‌کند
  متغیرهای انزوای کامل — اما که *هر* ابزار آگاه XDG را تأثیر می‌گذارد
  شما در پروژه اجرا می‌کنید. `doctor` هر دو وضعیت را گزارش می‌دهد.
- **پروژه `.claude/` رفتار بومی Claude است.** Claude Code همیشه
  `./.claude/` را می‌خواند صرف‌نظر از `CLAUDE_CONFIG_DIR`. agentmod اضافی شده ارزش برای Claude
  منزوی کردن حالت *سطح کاربر* است (مهارت‌های جهانی/افزونه‌ها،
  جلسات، تاریخچه)؛ پروژه `.claude/` قبل از agentmod کار کردند.
- **فعال‌سازی هوک جلسه اول.** درست بعد از `agentmod init`، shell
  اجرایی موجود بلوک rc جدید را بارگذاری نکردند. ترمینال جدیدی باز کنید،
  `exec $SHELL`، یا یک‌بار `eval "$(agentmod hook zsh)"` (init
  دقیقاً چاپ می‌کند). همچنین، هوک bash از `PROMPT_COMMAND`
  و بنابراین غیرفعال در bash اسکریپت‌های غیر تعاملی (همان کلاس محدودیت دیگر
  direnv) — اسکریپت‌ها متغیرها را صریح تنظیم کنید توسط
  `eval "$(agentmod env --shell bash --activate <root>)"` اگر آن‌ها
  مسیریابی لازم می‌دارند.
- **تنها npm bin جهانی در PATH است.** `.agentmod/node/bin` یک
  ورودی PATH مدیریت شدند است. نصب‌های جهانی pnpm/bun درون پروژه مسیریابی شده‌اند
  (`PNPM_HOME`، `BUN_INSTALL`) اما bin دایرکتوری‌های آن‌ها PATH اضافه نمی‌شوند.
- **بسته‌های درخت بازگردانی‌های دستی.** `handoff restore` فقط `.amod` فایلها را می‌پذیرند؛ یک
  دایرکتوری `agentmod-handoff./` عهدنامه توسط
  دنبال کردن `RESTORE.md` درون آن بازگردانیده می‌شود (این نسخه بدون
  خوانندگی دایرکتوری است).
- **تصاویر منطبق ممکن است ترمیم بعد از بازگردانی نیاز دارند.** کلون gstack بدون
  `.git` سفر می‌کند (دوباره اجرا `agentmod install gstack --force` برای ساخت آن
  به‌روز‌پذیر دوباره)، و symlink‌های آغازگر `node/bin` آویزان می‌شوند زیرا
  `node_modules` مستثنی است (دوباره اجرا `npm install -g …` درون
  پروژه).
- **پشتیبانی Shell برای zsh و bash است.** پوسته‌های دیگری می‌توانند هنوز استفاده کنند
  `agentmod env` دستی.

## سؤالات متداول

**آیا من به `claude` / `codex` / `opencode` مستقیم استفاده می‌کنم؟**
بلی. آن است نقطه — بدون بسته‌بندی، بدون shims، بدون `agentmod run`.

**چرا agentmod فقط `HOME` را تغییر نمی‌دهد؟**
مجدداً اختصاص `HOME` SSH، git، keychains، dotfiles، و هر ابزار دیگر
در پوسته شکست می‌خورد. agentmod فقط متغیرهای مختص عامل را مسیریابی می‌کند.

**چرا احراز هویت من بعد از بازگردانی غایب است؟**
طرح‌ریزی شده — اعتبار هرگز در تصاویر منطبق سفر نمی‌کند. خطوط دوباره ورود چاپ شده را
دنبال کنید (یا تجویز init کپی) در ماشین جدید.

**آیا من `.agentmod/` را به git فهرست کار دهم؟**
نه — init آن gitignores (جلسات، حافظه پنهان، و احتمالاً اعتبار کپی شده
زندگی آنجا). بجائے آن بسته ایمن: `agentmod pack --for-git`.

**چطور این متفاوت از direnv است؟**
مدل فعال‌سازی یکسان (دایرکتوری محدوده شده env، prompt-hook مبتنی، بازگردانی کامل
عملیات خروج)، اما agentmod همچنین می‌داند *چه* برای هر عامل مسیریابی‌کند،
خانه‌ها را ایجاد می‌کند، نوشتار جهانی را حفاظت می‌کند، و تحویل انجام می‌دهد. دو
همزیستی خوبی دارند.

**یک تصویر منطبق با "secret-candidate findings" ایجاد شکست می‌خورد.**
اسکن محتوای ماده کلید خصوصی در یک فایل نگهداری شده پیدا کردند. آن را حذف کنید (یا
آن را به مکان مستثنی جابجا کنید مثل `.env`)، یا بجائے آن بسته کنید
`--allow-findings` اگر آن را درون تصویر منطبق قبول می‌کنید.

**آیا این در Windows کار می‌کند؟**
کد Go ساخته می‌شود و ایمنی مسیر برای مسیرهای سبک Windows اعمال می‌شود، اما
هوک shell برای zsh/bash؛ Windows در این نسخه نازمایش‌شده است.
